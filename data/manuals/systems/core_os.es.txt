RESUMEN DE SISTEMA — core_os
Ref: OS-MAN/SYS/CORE-OS

core_os es la capa mínima de supervisión de la nave. Si falla, pierdes autoridad de mando.

Función
- Planifica jobs, enruta alertas y mantiene sesión del operador.
- Proporciona la interfaz base de comandos.

Síntomas de degradación
- Telemetría incompleta, retrasos en tareas, registros corruptos.

Impacto por estado
- NOMINAL/UPGRADED: control completo (si las demás condiciones del sistema lo permiten).
- LIMITED: bloquea comandos avanzados (`uplink`, `route solve`, `auth recover`).
- DAMAGED: bloquea lo mismo que LIMITED y además bloquea viaje (`nav <node_id>`, `nav --no-cruise <node_id>`).
- CRITICAL: set de recuperación de emergencia.
- OFFLINE: bloqueo de terminal; solo set mínimo de supervivencia.

Set mínimo de supervivencia (terminal)
- `help`, `clear`, `exit`.
- `status`.
- `alerts`, `alerts explain <alert_key>`.
- `logs`, `log copy [n]`.
- `jobs`, `jobs <n|all>`.
- Debug de soporte: `debug on|off|status`, `debug arcs`, `debug lore`, `debug deadnodes`, `debug modules`, `debug seed <n>`, `debug scenario <preset>`.

Comandos relacionados
- `status`, `alerts`, `logs`, `diag core_os`.
