SYSTEM SUMMARY — data_core
Ref: OS-MAN/SYS/DATA-CORE

`data_core` is the persistent data plane for manifests, recovery workflows, and structured telemetry.

Function
- Supports cargo manifest synchronization and audit jobs.
- Hosts credential recovery workflows.
- Supplies telemetry/data services to uplink and survey operations.

Operational dependencies
- Primary service daemon: `datad`.
- Most data operations require `data_core >= limited` and running `datad`.

Command impact by state
- `cargo audit` / `inventory audit`: require `data_core >= limited` + `datad`.
- `auth recover`: requires `data_core >= limited` + `datad` (+ power gating).
- `uplink`: requires `data_core >= limited` + `datad` (+ dock/node/power gating).
- `ship survey`: requires `data_core >= limited` + `datad` plus sensor stack availability.

Common symptoms
- stale manifests, failed auth recovery jobs, reduced intel ingestion.

Recovery pattern
- repair `data_core`,
- boot `datad`,
- rerun blocked data workflows.

Related commands
- `boot datad`, `cargo audit`, `auth recover`, `uplink`, `ship survey`
