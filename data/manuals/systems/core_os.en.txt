SYSTEM SUMMARY — core_os
Ref: OS-MAN/SYS/CORE-OS

`core_os` is the ship supervisory kernel. If it collapses, command authority collapses with it.

Function
- Schedules jobs and orchestrates command execution.
- Routes alerts and event streams to operator interfaces.
- Maintains active command session and control permissions.

Operational dependencies
- Requires stable baseline power and minimal service continuity to preserve full command set.
- In severe degradation, command availability is reduced to survival operations.

Command impact by state
- NOMINAL/UPGRADED: full command surface (subject to subsystem-specific gating).
- LIMITED: blocks advanced ops (`uplink`, `route solve`, `auth recover`).
- DAMAGED: LIMITED restrictions plus travel commands blocked.
- CRITICAL: emergency-recovery command set only.
- OFFLINE: terminal lockout; minimal survival command subset.

Common symptoms
- delayed jobs, incomplete telemetry, fragmented logs, command rejection drift.

Recovery pattern
- restore power stability,
- prioritize `core_os` and `energy_distribution` repairs,
- recover blocked services in dependency order.

Related commands
- `status`, `diag core_os`, `alerts`, `logs`
