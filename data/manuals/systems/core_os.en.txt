SYSTEM BRIEF — core_os
Ref: OS-MAN/SYS/CORE-OS

core_os is the ship’s minimal supervisory layer. If it fails, you lose command authority.

Role
- Schedules jobs, routes alerts, and maintains operator session.
- Provides the base command interface.

Degradation symptoms
- Missing telemetry, delayed jobs, corrupted logs.

State impact
- NOMINAL/UPGRADED: full control (if other system conditions allow).
- LIMITED: blocks advanced commands (`uplink`, `route solve`, `auth recover`).
- DAMAGED: blocks the same as LIMITED and also blocks travel (`nav <node_id>`, `nav --no-cruise <node_id>`).
- CRITICAL: emergency recovery command set.
- OFFLINE: terminal lock; minimal survival set only.

Minimal survival terminal set
- `help`, `clear`, `exit`.
- `status`.
- `alerts`, `alerts explain <alert_key>`.
- `logs`, `log copy [n]`.
- `jobs`, `jobs <n|all>`.
- Support debug: `debug on|off|status`, `debug arcs`, `debug lore`, `debug deadnodes`, `debug modules`, `debug seed <n>`, `debug scenario <preset>`.

Relevant commands
- `status`, `alerts`, `logs`, `diag core_os`.
