SYSTEM SUMMARY — sensors
Ref: OS-MAN/SYS/SENS

`sensors` is the local detection and acquisition layer for contacts, routes, and survey telemetry.

Function
- Feeds local discovery and scan operations.
- Supports route solving within effective range.
- Contributes sensor-side telemetry to survey workflows.

Operational dependencies
- Boot dependency: `energy_distribution >= nominal`.
- Service daemon: `sensord` (must be running for active sensor workflows).

Command impact by state
- `scan`: requires `sensors >= limited` + `sensord` running.
- `route solve`: requires `sensors >= limited` + `sensord` running (+ power/range gating).
- `ship survey`: requires sensors stack plus data stack availability.
- OFFLINE or stopped `sensord`: no active sensor telemetry.

Common symptoms
- route solve interruptions, empty scans, degraded situational awareness.

Recovery pattern
- repair `sensors`,
- boot `sensord`,
- reattempt blocked scan/solve operations.

Related commands
- `boot sensord`, `diag sensors`, `scan`, `route solve`, `ship survey`
