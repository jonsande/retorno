Purpose
- Unified command family for drone deployment, movement, recovery, salvage, and module operations.

Requirements (gating)
- Deploy/recall/reboot/repair/salvage require a valid drone and battery/integrity gates.
- World-node operations require drone and ship to remain co-located at that node.
- `drone install <drone_id> <module_id>` resolves by module scope:
  - if `scope=ship`, requirements: operator drone deployed on ship, module in inventory, scrap cost available.
  - if `scope=drone`, requirements: `docked` in `drone_bay`, `drone_bay=NOMINAL`, `energy_distribution!=OFFLINE`, free slots.
- `drone uninstall <drone_id> <module_id>` resolves by module scope:
  - if `scope=ship`, requirements: operator drone deployed on ship, module installed on ship, bay support online.
  - if `scope=drone`, requirements: `docked` in `drone_bay`, `drone_bay=NOMINAL`, `energy_distribution!=OFFLINE`.

Operational behavior
- Drone modules do not mutate base stats destructively.
- Effective profile is derived from base chassis + installed modules.
- Effective profile modifies ETAs for `move`, `deploy/deploy!`, `survey`, and active `repair`.
- Repair failure uses the same core failure model, with effective multipliers from operator drone modules.
- Docked maintenance (charge/passive repair) respects effective maxima (`battery_max_effective`, `integrity_max_effective`).
- `drone salvage scrap` ETA scales with requested load vs drone effective cargo capacity.

Common block reasons
- `drone_not_in_bay` for drone-module install/uninstall when target drone is not docked in bay.
- `drone_bay_install_offline` when bay support is offline.
- `drone_bay_not_nominal` when drone-module operation is requested outside NOMINAL bay state.
- `energy_distribution_offline` when bay module operation is attempted with offline distribution.
- `module_scope_mismatch` when operation and module scope are incompatible.
- `module_slots_full` when target drone has no free module slots.
- `drone_module_not_installed` when uninstalling a module not present on target drone.
- `module_not_installed` when uninstalling a ship module that is not currently installed.

Operational notes
- Base drone slots in v1: 2.
- All v1 drone modules consume 1 slot.
- Recovered drones may include deterministic preinstalled drone modules.

Related commands
