module inspect <module_id>

Purpose
- Inspect one installed module and its operational effects.

Command family status
- `module` and `modules` share the same installed-module context.
- Installation workflow is external to this command family and is drone-operated.

Operational behavior
- `module inspect <module_id>` shows module effects and description/lore notes.
- If the module is not installed, inspect will report unavailable context.

Related commands
- `modules`, `drone install <drone_id> <module_id>`, `debug modules`
