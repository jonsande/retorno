modules

Purpose
- Show module state split by installation scope and runtime placement.

Operational behavior
- `modules` displays three views:
  - ship installed modules,
  - drone installed modules grouped by `drone_id`,
  - inventory modules grouped by scope (`ship`/`drone`).
- For drones, slot occupancy is shown against slot capacity.
- Inventory section uses manifest view; if manifest is stale, run `cargo audit`.

Related commands
- `module inspect <module_id>`, `drone install <drone_id> <module_id>`, `drone uninstall <drone_id> <module_id>`, `cargo audit`
