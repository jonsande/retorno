modules

Purpose
- List currently installed modules on RETORNO.

Command family behavior
- `modules` focuses on installed inventory only.
- Alias: `module`.

Operational notes
- If list is empty, install via drone workflow.
- For module-level detail, use `module inspect <module_id>`.
- For full catalog visibility (installed + not installed), use debug module listing.

Related commands
- `module inspect <module_id>`, `drone install <drone_id> <module_id>`, `debug modules`
