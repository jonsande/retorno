uplink

Propósito
- Establecer un enlace de datos con el relay/station/waystation actual para ingerir inteligencia de navegación.

Requisitos (gating)
- La nave debe estar acoplada en un tipo de nodo elegible.
- `data_core >= limited` y `datad` en ejecución.
- Energía suficientemente estable para operación de enlace:
  - bloqueado con `Q < 0.55`,
  - bloqueado durante brownout,
  - bloqueado en estado crítico de energía.

Comportamiento
- Descarga intel de rutas/contactos desde cachés remotas.
- Actualiza enlaces conocidos y puede abrir nuevos caminos alcanzables.
- Escribe artefactos de log de navegación en `/logs/nav/`.
- Los datos de uplink de un nodo son de un solo uso: tras una descarga exitosa en ese nodo, repetir `uplink` allí no devuelve datos nuevos.
- El pool de rutas de uplink de un nodo queda fijado cuando ese nodo pasa a ser contacto conocido.

Bloqueos frecuentes
- `not_docked`
- tipo de nodo no elegible
- `deps_unmet` / servicio detenido
- `power_quality_low` / `brownout_active` / `critical_power_state`

Comandos relacionados
- `route solve <node_id>`, `nav map routes`, `intel`, `logs`
