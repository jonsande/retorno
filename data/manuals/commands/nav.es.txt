Descripción
- `nav map sectors`: lista sectores del mundo conocidos y coordenadas de sector.
- `nav map galaxy [escala]`: muestra el mapa galáctico en la escala seleccionada (`sector=10ly`, `local=100ly`, `regional=10000ly`, `global=500000ly`) usando nodos conocidos + posición actual.
- `nav map graph`: muestra el grafo global de nodos/rutas conocidas.
- `nav map graph <node_id>`: enfoca el grafo en un nodo conocido.
- `nav map path <node_id>`: calcula la mejor ruta conocida desde el nodo actual.
- `nav map routes`: lista rutas directas conocidas desde el nodo actual.
- `nav map contacts`: lista contactos conocidos con sector, región y estado de ruta relativo al nodo actual.
- `nav map contacts sector`: agrupa los contactos conocidos por sector y ordena los sectores por cercanía.
- `nav <node_id>`: inicia tránsito al destino.
- `nav --no-cruise <node_id>`: inicia tránsito sin forzar perfil CRUISE.
- `nav abort`: aborta tránsito en curso y revierte al origen.

Aliases aceptados (no aparecen en `help`)
- `nav sectors` -> `nav map sectors`
- `nav galaxy` -> `nav map galaxy`
- `nav galaxy <scale>` -> `nav map galaxy <scale>`
- `nav graph` y `graph` -> `nav map graph`
- `nav graph <node_id>` y `graph <node_id>` -> `nav map graph <node_id>`
- `map path <node_id>` y `path <node_id>` -> `nav map path <node_id>`
- `nav routes` y `routes` -> `nav map routes`
- `nav contacts` y `contacts` -> `nav map contacts`
- `nav contacts sector` y `contacts sector` -> `nav map contacts sector`
- `travel <node_id>` -> `nav <node_id>`

Dependencias y gating
- `nav map *`: depende solo de `core_os`.
- `nav <node_id>` y `nav --no-cruise <node_id>`: dependen de `core_os` y de que exista ruta/viaje válido.
- Cap duro de salto de tránsito: cualquier salto por encima de `MAX_ROUTE_HOP_LY` se bloquea (por defecto `45ly`).
- Estado `core_os`:
  - `NOMINAL/UPGRADED`: control completo.
  - `LIMITED`: bloquea `uplink`, `route solve`, `auth recover`.
  - `DAMAGED`: bloquea lo anterior y también `nav <node_id>` y `nav --no-cruise <node_id>`.
  - `CRITICAL`: solo set de recuperación de emergencia.
  - `OFFLINE`: terminal mínima de supervivencia.

Comandos relacionados
- `route solve <node_id>`, `dock <node_id>`, `undock`, `hibernate until_arrival`, `intel`, `debug galaxy map <scale>`
