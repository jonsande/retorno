uplink

Purpose
- Establish a data uplink with the current relay/station/waystation to ingest navigation intelligence.

Requirements (gating)
- Ship must be docked at an eligible node type.
- `data_core >= limited` and `datad` running.
- Power must be stable enough for data-link operations:
  - blocked when `Q < 0.55`,
  - blocked during brownout,
  - blocked in critical power state.

Operational behavior
- Pulls route/contact intel from remote caches.
- Updates known links and may unlock new reachable paths.
- Writes navigation log artifacts under `/logs/nav/`.

Common block reasons
- `not_docked`
- ineligible node type
- `deps_unmet` / service not running
- `power_quality_low` / `brownout_active` / `critical_power_state`

Related commands
- `route solve <node_id>`, `nav map routes`, `intel`, `logs`
