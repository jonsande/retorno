undock

Purpose
- Release the ship from a docked node and return to local orbital posture.

Requirements (gating)
- Ship must currently be docked.
- Action must pass current power-state restrictions.

Behavior
- Undock queues a timed job (`Balance.UNDOCK_TIME_S`).
- Job can be interrupted if dock state changes during execution.
- `job cancel <job_id>` can abort the active undock job.

Common block reasons
- `not_docked`
- `critical_power_state`
- context changed before completion (job interruption)

Related commands
- `dock <node_id>`, `jobs`, `job cancel <job_id>`, `status`
